### AWS re/Start Learning Journey

### Week 6 – AWS Security Services

####  Topics & Services Covered
- AWS IAM — Identity and Access Management Foundations
- AWS IAM Identity Center & Access Analyzer
- AWS Security Token Service (STS)
- Amazon GuardDuty — intelligent threat detection
- Amazon Inspector — automated vulnerability assessment
- Amazon Macie — sensitive data discovery in S3
- Amazon Detective — security investigation and analysis
- AWS Security Hub — centralised security findings
- AWS Encryption Fundamentals (KMS)
- AWS Config — resource configuration compliance
- Amazon CloudWatch — monitoring and observability
- AWS CloudTrail — API activity auditing
- AWS Trusted Advisor
- AWS Organizations
- AWS Systems Manager
- AWS Managed Services: Security & Patch Management

####  Labs & Hands-On Projects
- Configured IAM users, groups, roles, and least-privilege policies
- Reviewed GuardDuty findings and threat intelligence feeds
- Studied CloudWatch metrics, alarms, and log groups
- Traced API activity using CloudTrail event history

####  Challenges & How I Solved Them
- **Challenge:** Distinguishing between CloudWatch, CloudTrail, and AWS Config — all monitoring-related but serving different purposes.
- **Solution:** Memorised this distinction — CloudWatch = *performance metrics and alarms*; CloudTrail = *who did what and when (API audit logs)*; Config = *is my infrastructure compliant with the rules I defined?*

####  Key Takeaways & Reflections
> Security in AWS is multi-layered — from identity (IAM) to threat detection (GuardDuty) to compliance (Config) to investigation (Detective). The principle of least privilege is not just a best practice; it is the foundation of a secure cloud environment. This was the most content-rich and practically important week of the programme.
