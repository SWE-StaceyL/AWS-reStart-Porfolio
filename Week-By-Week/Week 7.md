### AWS re/Start Learning Journey

### Week 7 – AWS Management Tools

####  Topics & Services Covered
- AWS CloudFormation — Infrastructure as Code (IaC)
- CloudFormation Stacks — troubleshooting and drift detection
- AWS Config — resource compliance tracking
- AWS Systems Manager — operational management at scale
- AWS Control Tower — multi-account governance and landing zones
- AWS Organizations — account management at enterprise scale
- AWS Trusted Advisor — cost, security & performance recommendations
- AWS Service Catalog — approved product portfolios
- Management Tools Cheat Sheets

####  Labs & Hands-On Projects
- Created and deployed CloudFormation stacks from templates
- Explored drift detection and stack troubleshooting in the console
- Reviewed Systems Manager Session Manager for secure, agentless instance access
- Studied Control Tower landing zones and preventive guardrails

####  Challenges & How I Solved Them
- **Challenge:** Troubleshooting failed CloudFormation stack deployments.
- **Solution:** Learned to read CloudFormation Events in the console to identify the root cause resource, and used rollback settings strategically — preserving failed stacks in development to inspect errors, enabling rollback in production for stability.

####  Key Takeaways & Reflections
> Infrastructure as Code (IaC) with CloudFormation transforms manual, error-prone deployments into repeatable, version-controlled templates. Management tools like Control Tower and Organizations show how enterprise-scale AWS environments maintain governance and compliance across hundreds of accounts consistently.
