

### Week 1 – Introduction to AWS & Cloud Foundations

####  Topics & Services Covered
- What is Cloud Computing? — deployment models, service models (IaaS, PaaS, SaaS)
- What is AWS? — global infrastructure, regions, and availability zones
- AWS Well-Architected Framework — the six pillars
- Shared Responsibility Model — AWS vs customer responsibilities
- Job Roles in the Cloud
- AWS Billing and Cost Management (Parts 1 & 2)
- AWS Support Plans

####  Labs & Hands-On Projects
- Explored the AWS Console and navigated core service categories
- Reviewed the AWS Pricing Calculator and billing dashboards
- Studied real-world examples of the Shared Responsibility Model

####  Challenges & How I Solved Them
- **Challenge:** Understanding the distinction between the six pillars of the Well-Architected Framework.
- **Solution:** Created a personal summary mapping each pillar to a real-world cloud scenario, which helped cement the differences.

####  Key Takeaways & Reflections
> Cloud computing is not just about moving servers online — it is a fundamental shift in how businesses think about infrastructure, cost, and scalability. The Shared Responsibility Model was an eye-opener: security is a *shared* effort, not something AWS handles alone. This week set the foundation for everything that followed.
